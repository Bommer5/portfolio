Conférence — Cloud avec M365 et Azure Infrastructure

Conférence 2h Azure · M365 · Cloud

Contexte

Le 3 novembre 2025, l'EPHEC a organisé une conférence intitulée "Cloud avec M365 et Azure Infrastructure", dans le cadre des activités d'enrichissement proposées aux étudiants en Technologies de l'Informatique. L'intervenant était Kevin Keurvels, consultant spécialisé Microsoft chez Axentys, une société belge reconnue pour son expertise en solutions cloud Microsoft. La conférence a duré 2 heures et j'ai reçu une attestation de participation officielle, signée par Marie-Noël Vroman (coordinatrice EPHEC).

Faits et apprentissages

Kevin Keurvels a présenté les technologies cloud Microsoft dans le contexte de leur utilisation réelle en entreprise, avec des exemples concrets et des retours d'expérience tirés de ses missions chez Axentys. Les principaux points abordés :

  • Architecture Azure : Présentation des services fondamentaux d'Azure — machines virtuelles, stockage Blob, App Services, Azure Functions — et comment ils s'articulent pour construire des infrastructures scalables.
  • Microsoft 365 en entreprise : Comment M365 (Teams, SharePoint, Exchange Online, OneDrive) s'intègre dans l'environnement de travail quotidien des grandes organisations, avec une gestion centralisée des identités via Azure Active Directory.
  • Sécurité et conformité cloud : Les enjeux de sécurité liés à la migration vers le cloud — gestion des accès, politiques de conformité, chiffrement des données — et les outils Microsoft dédiés (Microsoft Defender, Purview).
  • Solutions hybrides : Comment de nombreuses entreprises combinent infrastructure on-premise et cloud (hybride) pendant leur migration, et les défis techniques que cela implique.

Mise en perspective

Cette conférence a significativement élargi ma vision du métier de développeur. Jusqu'ici, je me focalisais essentiellement sur le code — le front-end, le back-end, les APIs. Mais Kevin Keurvels a mis en évidence que dans une grande entreprise, le code n'existe pas en vase clos : il tourne dans des environnements cloud gérés par des équipes infrastructure, sur des identités gérées par Active Directory, dans des contextes de conformité réglementaire stricts.

Cette prise de conscience répond directement à l'une de mes faiblesses identifiées : la méconnaissance de l'aspect infrastructure et "ops" du développement. J'ai compris que pour travailler efficacement dans une grande entreprise IT, un développeur full-stack doit avoir au minimum une culture cloud — même s'il n'est pas lui-même administrateur système.

Lien avec mon projet professionnel

AXA, comme la quasi-totalité des grandes entreprises belges du secteur financier, s'appuie massivement sur Microsoft Azure et les outils M365. Les développeurs d'AXA déploient leurs applications sur Azure, utilisent Azure DevOps pour la gestion de projets et les pipelines CI/CD, et travaillent dans un environnement M365 au quotidien.

Avoir assisté à cette conférence et compris les fondamentaux de cet écosystème constitue donc un avantage direct et concret pour mon intégration dans cet environnement. Je sais maintenant ce qu'est une App Service Plan, pourquoi Azure Active Directory est central dans la gestion des identités d'entreprise, et quel rôle joue Microsoft Defender dans la sécurité cloud. Ce sont des connaissances qui me permettront de ne pas être totalement perdu lors de mes premiers jours en entreprise.
